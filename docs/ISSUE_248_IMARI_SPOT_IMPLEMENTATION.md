## 既存広域地点と移行方針

両地点は `imari-inner-bay` の広域文脈に含まれる具体地点として追加する。既存IDは削除、改名、非active化せず、既存釣果、地点詳細、環境キャッシュ、履歴状態を推測で再割当しない。migrationはforward-onlyで、remote Supabaseへは適用しない。復旧が必要な場合は後続migrationで新規地点を非active化し、既存レコードを削除しない。
