# データ利用ポリシー

## 目的

このプロジェクトでは、将来的に釣具店、ブログ、SNS、公式発表、ユーザー投稿などの釣果情報を扱う可能性があります。

MVPでは、法務リスク、著作権リスク、アクセス負荷の問題を避けるため、モックデータのみで開始します。

## 基本方針

- 各サイトの利用規約を尊重する。
- 必要に応じてrobots.txtを確認し、尊重する。
- 著作権に配慮する。
- 不要なアクセス負荷をかけない。
- 公式API、RSS、許可を得た情報源、ユーザー提供の本文/URLを優先する。
- 保存するのは抽出した事実情報と出典URLを中心にする。
- 出典を明示する。
- アプリが釣果を保証するような表現はしない。

## MVP v0.1 で許可すること

- モック釣果データの利用。
- 出典名と出典URLのフィールド定義。
- 釣れそう度スコアと根拠のモック表示。

## MVP v0.1 で禁止すること

- 外部釣果サイトの自動取り込み。
- 記事URL貼り付け機能。
- 高頻度の自動取得。
- 第三者サイトの記事本文の再掲載。
- 第三者サイトの画像コピー。
- アクセス制限の回避。
- 明示的な承認なしの有料データAPI利用。

## 将来の情報源候補

将来、釣果情報の取り込みを検討する情報源候補は以下です。

- まるきん。
- 釣具のポイント。
- 福岡北西部から佐賀県西部の店舗情報。

## 将来の取り込み方針

釣果情報の取り込みを追加する場合は、以下の優先順位で検討します。

1. 公式API。
2. RSSフィード。
3. 許可を得た情報源。
4. ユーザーが提供したURL/本文からの抽出。
5. 法的・技術的に問題がない範囲での低頻度取得。

## 保存するデータ項目

保存対象は、以下のような構造化された事実情報を中心にします。

- 釣果日。
- エリア名、または釣り場名。
- 緯度。
- 経度。
- 魚種。
- 釣果数。
- サイズcm。
- 釣り方。
- 出典名。
- 出典URL。
- 作成日時/更新日時。

保存を避けるものは以下です。

- 第三者サイトの記事本文全文。
- 第三者サイトの画像。
- アプリに不要な個人情報。
- 同意のない正確なユーザー現在地。

## 地点情報に関する注意

このアプリは個人利用を前提とします。MVPでは代表地点の緯度・経度を扱います。

将来公開する場合は、釣り場保護やトラブル防止のため、地点座標の丸め、詳細地点の非表示、公開範囲の制限を検討します。

## 注意書き文言

アプリ上では、以下のような注意書きを表示します。

> 釣れそう度は、取得可能な釣果情報と簡易ルールに基づく参考情報です。実際の釣果を保証するものではありません。

## Post-MVP-002 で扱う外部環境データ

Post-MVP-002では、無料・APIキーなしで利用できるOpen-Meteo Weather Forecast APIとOpen-Meteo Marine Weather APIを、天気・海況の参考表示に利用します。

- 外部釣果サイトの自動取り込みは行いません。
- `https://tide.chowari.jp/` は潮見表・タイドグラフ・表示項目・UI構成の参考に留め、スクレイピング、コピー、転載、取得元としての利用は行いません。
- Marine APIの `sea_level_height_msl` は潮位そのものを保証する値ではなく、潮位参考値として表示します。
- Open-Meteoの値は予報値・参考値であり、航海用・安全判断用には利用しない旨をUIに表示します。
- APIキーやシークレットは追加しません。

## Post-MVP-002.5 で扱う地図タイルと出典表示

Post-MVP-002.5では、航空写真表示の候補として国土地理院の地理院タイル `modis`、`lndst`、`seamlessphoto` を利用します。

- 低倍率タイルURL: `https://cyberjapandata.gsi.go.jp/xyz/modis/{z}/{x}/{y}.png`（世界衛星モザイク画像、ズーム2〜8）
- 中倍率タイルURL: `https://cyberjapandata.gsi.go.jp/xyz/lndst/{z}/{x}/{y}.png`（全国ランドサットモザイク画像、ズーム8〜14）
- 高倍率タイルURL: `https://cyberjapandata.gsi.go.jp/xyz/seamlessphoto/{z}/{x}/{y}.jpg`（全国最新写真、ズーム14〜18）
- MapLibre GL JSの `raster` source / `raster` layerとして表示し、航空写真モード時のみ表示します。
- Google Maps / Google航空写真は、この段階では利用しません。
- 航空写真表示中は、地図付近またはマップ内に `出典: 国土地理院ウェブサイト / 地理院タイル` を表示します。
- UI上では `低倍率では衛星モザイク画像、中〜高倍率ではランドサット/全国最新写真を表示します。` と注記します。
- 世界衛星モザイク画像と全国ランドサットモザイク画像は、国土地理院の出典表示に加えて元データの追加出所明示が必要になる可能性があるため、公開前に最新の提供条件、出典記載方法、利用上の制約を確認します。

釣り場マスターの座標は、MVP検証用の代表点として扱います。公開範囲を広げる場合は、釣り場保護、混雑防止、安全配慮のため、詳細座標の丸め、非公開化、危険な小場所の除外を検討します。

## Post-MVP-003: 外部釣果情報の保存方針

外部釣果情報は、将来の `SCORE` 改善に使える可能性がありますが、MVP後の検証段階では法務リスク、著作権リスク、アクセス負荷を避けることを優先します。

保存する情報は、記事本文や画像そのものではなく、釣果を説明するために必要な構造化データと出典URLに限定します。

### 保存する情報

- 魚種。
- 釣果日。
- エリア名。
- 推定釣り場名。
- 釣り場マスターに紐づけできる場合の `spotId`。
- 緯度・経度を扱う場合の座標精度区分: `exact` / `approximate` / `rounded` / `unknown`。
- 釣り方。
- 匹数。
- サイズcm。
- 情報元ID、情報元名、出典URL。
- 取得方法: `manual` / `ai_assisted` / `auto`。
- 信頼度: `high` / `medium` / `low`。
- 天気、潮汐、海況などとの照合メモ。
- 作成日時/更新日時。

### 保存しない情報

- 第三者サイトの記事本文全文。
- 第三者サイトの記事本文の長文転載。
- 第三者サイトの画像コピー。
- コメント全文。
- アプリに不要な個人情報。
- 同意のない正確なユーザー現在地。

### 座標精度区分

外部情報は地点が曖昧な場合があるため、座標を保存する場合は必ず精度区分を持たせます。

- `exact`: 情報元または許可済みデータから具体地点として扱えるもの。
- `approximate`: 記事や投稿から推定した代表地点。
- `rounded`: 釣り場保護、混雑防止、公開範囲調整のため丸めた地点。
- `unknown`: 地点が不明、またはエリア名のみで扱うもの。

個人利用段階では詳細座標のぼかしは必須ではありませんが、公開範囲を広げる場合や外部情報が曖昧な場合に備えて、精度区分は維持します。

## 外部釣果情報の自動収集前チェックリスト

外部サイトへの自動収集を実装する前に、情報元ごとに以下を確認します。

- 利用規約を確認したか。
- robots.txtを確認したか。
- 取得対象ページの構造を確認したか。
- 公式API、RSS、許可を得た情報源、ユーザー提供URL/本文など、より安全な手段を優先できないか。
- 本文や画像をコピーせず、構造化データと出典URLだけを保存できるか。
- 出典URL、情報元名、取得方法を必ず保持できるか。
- アクセス頻度が過剰でないか。初期案は1日1回程度に留める。
- サイト運営者や投稿者に不利益を与える利用になっていないか。
- 自動収集不可、または判断が難しい場合の扱いを `manualOnly` または `referenceOnly` として決めたか。
- 取得データを直接確定扱いにせず、信頼度やレビュー状態を持たせる設計になっているか。

利用規約またはrobots.txt的に自動収集が難しいサイトは、自動収集対象から外し、URL手動登録または参照のみ扱いにします。

## 将来の `SCORE` 反映候補

外部釣果情報を `SCORE` に反映する場合は、釣果を保証する表現を避け、説明可能な参考要素として扱います。

反映候補は以下です。

- 同じ魚種の直近釣果があるか。
- 同じエリアの直近釣果があるか。
- 同じ釣り方の釣果があるか。
- サイズや匹数が良い釣果か。
- 釣果時点の天気、風、波、潮汐、海水温などの環境条件が現在または予測時点と近いか。
- 情報元の信頼度、取得方法、データ鮮度が十分か。

この段階では、外部釣果データを使った `SCORE` 再計算の本実装は行いません。
