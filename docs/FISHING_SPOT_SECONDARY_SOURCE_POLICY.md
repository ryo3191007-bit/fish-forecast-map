# 釣り場属性調査・二次情報源ポリシー

最終更新: 2026-07-12 JST  
対象: FishForecastMap Post-MVP-040以降の手動調査

## 1. 位置付け

この文書は `docs/FISHING_SPOT_RESEARCH_SPEC.md` を補足し、公式・公的情報だけでは確認できない釣り場属性を、民間情報や個人釣行記録からどの範囲まで整理できるかを定義します。

共通仕様の「民間釣りサイトは地点候補を知る入口に限定する」という原則は維持します。本ポリシーは、ユーザーが指定した個別地点について公式・公的調査を先に行ったうえで、不足属性だけを手動で補助調査する場合の限定ルールです。地域単位の一覧作成や、特定サイトのデータベース再現には適用しません。

本ポリシーは、手動で指定地点を調査する場合だけに適用します。外部サイトのスクレイピング、自動巡回、定期実行、API未許可サイトの自動取得を許可するものではありません。現時点で `crawlPolicy === "allowed"` の外部サイトが存在しない前提は変更しません。

## 2. 基本方針

1. 公式・公的情報を最初に確認する。
2. 不足項目だけを対象に、民間情報・個人ブログ・公開SNS等を手動で確認する。
3. 元ページの文章、攻略構成、魚種一覧、写真、地図画像、コメント、プロフィール情報を保存しない。
4. 保存するのは独自項目へ正規化した事実候補、根拠URL、確認日、確度、短い判定理由だけとする。
5. 民間情報1件のみの場合は原則 `confidence=low` とする。
6. 独立した複数の一般情報源が一致した場合は、内容に応じて `confidence=medium` とできる。
7. 規制・安全・施設利用については、肯定情報と否定情報を非対称に扱う。

## 3. 許可する使い方

- 特定地点について、複数の公開ページを手動で確認する
- 海底、水深、障害物、釣れる範囲等を低確度の推定材料として使う
- 最近の現地写真・動画・釣行記録から設備や構造物の存在候補を確認する
- 日付・地点・魚種が明記された具体的な釣果記録を、地点魚種の低確度観測として整理する
- 独立した複数sourceで一致した魚種を中確度の観測候補として整理する
- 非公式sourceに禁止・閉鎖・危険情報がある場合、低確度の警告として保持する

## 4. 禁止する使い方

- 特定サイトの釣り場一覧を地域単位で再現する
- 掲載魚種一覧を地点ごとに丸ごと転記する
- 攻略文や説明文を単純に言い換えて保存する
- 写真、地図画像、地図ピン、コメント、プロフィールを取得・保存する
- ログイン限定ページやアクセス制限を回避する
- 同一運営元の転載記事を複数sourceとして数える
- 釣り人が写っていることだけで現在も立入・釣り可能と判定する
- 夜釣り記録だけで常夜灯ありと判定する
- 駐車車両の写真だけで釣り目的の駐車場ありと判定する
- 情報が見つからないことから、禁止なし・閉鎖なし・障害物なしと判定する

## 5. 属性別の採用基準

### 海底・水深・潮通し

- 海図、港湾資料、研究資料がある場合はそれを優先する
- 民間情報1件だけの場合は `inferred / low` が上限
- 複数sourceが一致しても、対象岸壁や範囲が異なる場合は `unknown` を維持する
- 「深い」「潮通しが良い」等の感想だけで分類しない

### 河川の影響・外海の影響

- 公的地図で位置関係を確認したうえで `inferred` とする
- 河口からの距離だけで塩分や流量を断定しない
- 観測値がない場合、確度は最大でも `medium`

### 常夜灯・障害物・釣れる範囲

- 施設公式情報または複数時期の現地情報を優先する
- 港湾作業灯、航路標識灯、街路灯を常夜灯と自動的に扱わない
- 写真1枚で地点全体の障害物なしを断定しない
- 実釣記録や構造物情報から距離区分を推定する場合は `inferred`

### 駐車場・トイレ

- 施設が存在することと、釣り目的で利用可能なことを分ける
- 路上、空き地、店舗、フェリー利用者専用駐車場を、釣り人向け駐車場として登録しない
- 利用条件が確認できなければ `unknown`

### 釣り禁止・立入禁止・工事・閉鎖

- 公式・管理者・現地看板を最優先する
- 非公式sourceが禁止・閉鎖を明記する場合は、`confirmed / low` の警告候補として保持できる
- 禁止範囲や現行性が不明な場合は `note` に再確認必須と記載する
- 禁止情報が見つからないだけでは `no` にしない
- 釣り禁止と立入禁止を同一視しない

### 地点単位の魚種

次のどちらかを満たす魚種だけを `observed` 候補として登録します。

1. 独立した2件以上のsourceが同一地点で言及する
2. 日付・地点・魚種が明確な具体的釣果記録がある

確度の目安:

- 公式施設・公的観測: `high`
- 独立した一般source2件以上が一致: `medium`
- 日付付き釣果1件または個人記録1件: `low`

登録しない例:

- 民間釣りサイトの魚種一覧だけに載っている魚種を一括登録
- 生態的に生息可能というだけで `observed` にする
- 釣果投稿1件から恒常的な対象魚と断定する

## 6. `status` と `confidence`

`confirmed` は、登録したsourceがその内容を直接支えていることを表します。公式確定を意味しません。

例:

- 民間sourceが一部釣り禁止を明記
  - `status=confirmed`
  - `confidence=low`
- 公的地図の位置関係から河川の弱い影響を推定
  - `status=inferred`
  - `confidence=low` または `medium`

民間source由来の値は、source種別と弱点を `note` に明記します。

## 7. 保存しない情報

- 記事本文・説明文の長文または要約転載
- 写真・動画ファイル・地図画像
- コメント全文
- 投稿者名、プロフィール、アカウント情報
- 個人の正確な行動履歴
- サイト独自の評価文や攻略ノウハウ

## 8. 表示・SCOREへの反映

本調査JSONへ値を保存しただけでは、アプリ表示やSCOREへ自動反映しません。

後続Issueで、属性ごとに次を決めます。

- 必要な最低confidence
- ユーザー編集値との優先順位
- 古い情報の失効条件
- 規制・安全情報の警告表示
- SCOREへ反映してよい属性

特に `confidence=low` の値は、初期状態では参考表示または要確認情報として扱います。
