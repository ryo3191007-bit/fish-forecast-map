# MVPスコープ

## MVP v0.1 の目的

釣果予測マップアプリの最小構成を、モックデータで作成します。

目的は、データベース、外部データ取り込み、高度な予測、3D表示を追加する前に、地図を中心とした基本導線とUIを検証することです。

## 対象範囲

MVP v0.1では以下を対象とします。

- Next.js + TypeScriptのWebアプリ。
- MapLibre GL JSによる地図画面。
- 糸島西岸から唐津湾、伊万里湾、平戸方面までのモック釣果データ。
- 地図上の釣果地点マーカー。
- 釣果情報一覧。
- 釣果情報カード、または詳細表示に近いUI。
- 魚種フィルタ。
- 0〜100点の説明可能な簡易ルールベースの釣れそう度スコア。
- スコア根拠の表示。
- 釣れそう度は釣果を保証しないという注意書き。
- lint、typecheck、buildの基本CI。

## 対象外

MVP v0.1では以下を対象外とします。

- 釣果登録フォーム。
- ユーザー自身の釣果登録。
- 外部の釣果サイトの自動取り込み。
- 記事URL貼り付け機能。
- Supabase/DB連携。
- 潮汐・天気・水温データ連携。
- 3D海底地形表示。
- 実績ベースの高度な予測。
- ログイン/認証。
- 有料APIの利用。
- モバイルアプリとしてのリリース。

## 初期対象エリア

初期サンプルエリアは、福岡県糸島市西岸から唐津湾、伊万里湾、平戸方面までの沿岸・湾エリアとします。

代表地点は、志摩野北、芥屋、野北漁港、呼子、唐津東港、伊万里湾、平戸周辺などを仮置きします。

MVPではモックデータを使用しますが、代表地点には緯度・経度を持たせます。将来公開する場合は、地点座標の丸めや非公開化を検討します。

## MVPの成功基準

- ユーザーがアプリを開いて、何ができるアプリかすぐ理解できる。
- モック釣果情報を一覧と地図で確認できる。
- 魚種で絞り込みできる。
- 釣れそう度スコアと、その理由を確認できる。
- MapLibre GL JSを使った地図表示の土台ができている。
- 後続フェーズで、潮汐・天気・水温連携、3D海底地形表示、予測高度化に進める構成になっている。
