## 2. 最重要の作業ルール

### 2.1 基本フロー

1. ChatGPTが次の作業を一つだけ整理する。
2. GitHub Issueを作成する。
3. ユーザーが内容を確認する。
4. Codexを使う場合は、ユーザーの明示的な`Codex起動OK`を待つ。
5. Codex Botの👀、返信、Task URLのいずれかを確認してから「起動済み」と判断する。
6. PR作成後はChatGPTが差分・CI・Vercel・制約違反をレビューする。
7. 最終マージはユーザーが行う。
8. マージ確認後、ChatGPTが対応Issueを`completed`で閉じる。

### 2.2 絶対に守ること

- 必要な作業は一つずつ提示する。
- ユーザーの完了報告前に次作業を重ねない。
- PRをユーザーの代わりにマージしない。
- Codex起動前に親コミット、既存PR継承可否、想定消費を確認する。
- Codex外からCodex管理中のPRブランチを更新すると同期不能になる可能性がある。
- Codexへ「最新コメントを見て」だけを渡さず、修正要件を全文で渡す。
- CI greenだけでマージ可と判断せず、テストが実装本体を実行しているか確認する。
- ユーザーの明示許可なしにGitHubコメント、コミット、Issue/PR操作を行わない。

### 2.3 今回の事故と再発防止

PR #114のブランチへCodex外から引き継ぎ書を直接コミットしたため、Codexが既存PRへ同期できなくなりました。その後、置き換えPR #119が指定HEADではなく古い親から作成され、Codexタスクが2回失敗しました。

再発防止:

- PRの親関係を`compare_commits`で確認してからCodexを起動する。
- Codex管理中ブランチへ直接commitしない。
- 置き換えPRは新PR作成前に開始SHAを確認する。
- 残トークンが少ない場合はCodexを使わず、既存コードを手動修正する。

## 3. セキュリティ・外部情報ポリシー

GitHub、Issue、PR、チャット、ログへ貼らないもの:

- Supabase `service_role` key
- Supabase DB password / connection string
- `.env.local`
- 実トークン、実パスワード、管理者権限値

外部釣果情報:

- スクレイピングしない。
- 自動収集・定期実行しない。
- AIによる外部記事解析を追加しない。
- 現在`crawlPolicy === "allowed"`のサイトはない。
- 外部メモはユーザーが手動入力する。
- 保存対象は魚種、釣果日、エリア、推定地点、釣法、数、サイズ、参照元名/URL、acquisitionMethod、confidence等の要約メタデータのみ。
- 記事本文、画像、全文コメント、プロフィール詳細を保存しない。

潮汐・安全情報:

- Open-Meteoの`sea_level_height_msl`は潮位参考値で、公式満潮・干潮時刻ではない。
- 波・風・水深を安全判定へ変換しない。
- 水深・3D地形は航海、立入可否、避難、安全判断に使用しない。

## 4. Supabase / DBの状態

### 4.1 master data

Supabaseから次を読み取ります。

- `fish_species`
- `fishing_spots`
- `source_registry`

未設定、取得エラー、空データ時は静的データへfallbackします。

### 4.2 Auth / 自分の釣果記録

- メールOTP / magic linkログイン。
- 未ログイン時はlocalStorage。
- ログイン済みかつDB利用可能時はSupabaseへ保存。
- DB失敗時はlocalStorage fallback。
- DB行はowner-scoped。
- 削除は`is_deleted = true`の論理削除。
- 旧localStorage記録は自動移行せず、ユーザーが選択した記録だけ明示移行。
- 内部型・テーブル名に`ExternalCatchMemo` / `external_catch_memos`が残るのは後方互換のため。

### 4.3 migration

新しいDB変更の正本:

```txt
supabase/migrations/<timestamp>_<description>.sql
```

現在、手動実行すべきSQLはありません。`supabase/sql`の旧設計ファイルを新規migration代わりに使用しないでください。

## 5. 環境予報

Weather API:

- 気温、weather code、降水量、降水確率、風速、風向、突風

Marine API:

- 海面水温、潮位参考値、波高、波向、波周期、海流速度、海流方向

条件:

- `timezone=Asia/Tokyo`
- `forecast_days=7`
- Weather / Marineを独立取得
- 片方成功時はpartial表示
- 30分以内のcacheはfresh
- 24時間以内のcacheはAPI失敗時fallback
- 24時間超は使用しない
- offsetなし時刻はAsia/Tokyoとして解釈
- 地点切替時はAbortと`isActive`で古い応答を防止

環境値は現在SCOREへ反映していません。環境補正は別Issueで設計してください。

## 6. 水深・3D地形の現在状態

### 6.1 採用source

- 第一source: `GEBCO_2026 Grid 15 arc-second`
- データ由来: `GEBCO_2026 TID Grid`
- fallback: `NOAA NCEI ETOPO 2022 60 Arc-Second Bedrock`
- 海岸線補助: 国土地理院標準地図overlay、opacity `0.40`

GEBCO canonical:

- bounds: west `128.5` / south `32.5` / east `130.8` / north `34.0`
- shape: `552 x 360`
- pixel-centre registered
- DEM nodata: `-32767`
- DEM min/max: `-277 / 1346`
- DEM source SHA: `6824253a950edddc9c5c6e47a77eccbaae788b550f7885062aae238143622151`
- DEM values SHA: `59f02c67f79aa3edb61548ddd0dcb669880f6164ccc97eb8dd1a9fbfb0fd244b`
- TID nodata: `127`
- TID observed codes: `0/11/17/40/43/44`
- TID source SHA: `04462cc4ffeba5b55f7397ce0feebd97a0686ef360395c7286b29bbb852cda84`
- TID values SHA: `f39a3d090f387d124c1b5a10ecfff113f186b5f916ad2cc4001d5bebf2a70688`

### 6.2 表示・fallback

- GEBCO成功: GEBCO color / hillshade / contour / label / terrain。
- GEBCO metadata、tile、decode、terrain初期化失敗: ETOPOへ切替。
- ETOPOも失敗: 水深layer、terrain、GSI overlay、出典を解除し通常地図へ戻す。
- 同じerrorはdedupe。
- 遅れて届いた非表示sourceのerrorで状態を飛ばさない。
- 通常地図復帰時の表示: `水深データを読み込めなかったため通常地図へ戻しました`。

### 6.3 TID UI

- `public/bathymetry/gebco-2026/tid-crop.json`を自サイト内から読み込む。
- `moveend`で現在centerを再集計。
- 中心周辺17×17セル。
- nodata `127`を割合の分母から除外。
- bounds外、読込失敗、nodata-onlyは非致命的な案内。

### 6.4 GSI出典

公式タイル一覧:

`https://maps.gsi.go.jp/development/ichiran.html`

ZL5〜8の追加出典として、GEBCO Digital Atlas由来等深線、海上保安庁許可第292502号（水路業務法第25条）、VMAP0 shorelineを表示します。GEBCO正本の出典と混同しません。overlay OFF時はGSI sourceを追加せず、追加出典も表示しません。

### 6.5 生成・変換

通常生成:

```bash
npm run generate:bathymetry
```

公式NetCDFから正本を更新する手動変換:

```bash
python -m pip install netCDF4
node tools/bathymetry/convert-gebco-netcdf.mjs /local/gebco.nc /local/gebco_tid.nc
```

runtime/buildから外部GEBCO/NOAA取得は行いません。人工的な`Math.sin(...)*12`等の起伏を絶対に追加しません。
