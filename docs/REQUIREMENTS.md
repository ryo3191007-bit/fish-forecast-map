# 要件定義

## プロダクト概要

釣果予測マップは、福岡県糸島市西岸から唐津湾、伊万里湾、平戸方面までの陸っぱり釣り情報を地図上で確認し、どの場所で、いつ、どの魚種を狙うとよさそうかを検討するための個人利用Webアプリです。

このアプリは釣果を保証するものではありません。直近の釣果、季節性、場所、魚種、簡易スコアをもとに、参考情報を提供します。

## 想定ユーザー

- 個人利用を前提とする。
- 福岡県糸島市西岸から唐津湾、伊万里湾、平戸方面までの陸っぱり釣行を検討する人。
- 堤防・磯場などで、ルアーまたはエサ釣りを行う人。

公開サービスとしての利用は想定しません。

## 提供価値

- 対象エリアの釣果情報を地図上で確認できる。
- 魚種や場所で釣果情報を絞り込める。
- 0〜100点の釣れそう度スコアを確認できる。
- 釣れそう度スコアの理由を確認できる。
- 釣果情報の出典を確認できる。
- 将来的に、潮汐・天気・水温、3D海底地形、釣果実績を組み合わせた判断材料にできる。

## MVP v0.1 の機能要件

- アプリ概要と主要導線を表示するトップページを用意する。
- MapLibre GL JSを使った地図画面を用意する。
- モックデータを使って釣果地点マーカーを表示する。
- モックデータを使って釣果情報一覧を表示する。
- 釣果情報カードに、日付、エリア、場所、魚種、釣果数、サイズcm、釣り方、出典を表示する。
- 魚種で絞り込みできるようにする。
- 0〜100点の説明可能な簡易ルールベースの釣れそう度スコアを表示する。
- スコアの根拠を表示する。
- 釣れそう度は釣果を保証するものではない、という注意書きを表示する。

## MVP v0.1 では実装しない機能

- 釣果登録フォーム。
- ユーザー自身の釣果登録。
- 外部釣果サイトの自動取り込み。
- 記事URL貼り付け機能。
- Supabase/DB連携。
- 潮汐・天気・水温データ連携。
- 3D海底地形表示。
- 実績ベースの高度な予測。
- ログイン/認証。
- 有料APIの利用。

## 初期対象エリア

MVP v0.1では、以下の範囲から代表地点を仮置きします。

- 福岡県糸島市西岸。
- 志摩野北周辺。
- 芥屋周辺。
- 野北漁港周辺。
- 唐津湾周辺。
- 呼子周辺。
- 唐津東港周辺。
- 伊万里湾周辺。
- 平戸方面。

MVPでは代表地点の座標を持たせます。将来公開する場合は、地点座標の丸めや非公開化を検討します。

`青物` と `根魚` はカテゴリとして扱います。

## 釣果データ項目

MVPのモック釣果データでは以下を持たせます。

- 日付。
- 魚種。
- 場所名。
- エリア名。
- 緯度。
- 経度。
- 釣果数。
- サイズcm。
- 釣り方。
- 出典名。
- 出典URL。
- 釣れそう度スコア。
- スコア根拠。

釣果数は最初は数値のみで管理します。「好調」「ポツポツ」「渋い」などの表現は後続候補とします。

## 釣り方の粒度

MVPでは以下の粒度を基本とします。

- ジギング。
- キャスティング。
- コマセ。
- 泳がせ。
- サビキ。
- エギング。
- その他。

後続候補として、ライトゲーム、ショアジギング、フカセ、投げ釣りなどを追加できます。

## 非機能要件

- 既存のChatGPT Plus利用範囲を中心に、追加の月額コストをできるだけ抑える。
- 無料、またはオープンソースのライブラリを優先する。
- 見た目の良い地図UIを重視する。
- 将来的な水深レイヤーや3D海底地形表示を見据えた構成にする。
- 実装はシンプルで保守しやすい構成にする。
- 明示的な承認がない限り、有料APIを導入しない。
- シークレット情報をリポジトリに保存しない。
- データ利用ポリシーをドキュメントに明記する。

## 将来追加したい要件

優先順位は以下です。

1. 潮汐・天気・水温データ連携。
2. 3D海底地形表示。
3. 釣果予測スコアの高度化。
4. 釣具店等の釣果情報取り込み。
5. Supabase/PostgreSQLによるデータ永続化。

釣果情報の将来の情報源候補は、まるきん、釣具のポイントの福岡北西部から佐賀県西部の店舗情報です。
