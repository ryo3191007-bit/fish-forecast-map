# 外部釣果メモのSCORE反映ルール設計

## 目的

このドキュメントは、ユーザーが手動登録した外部釣果メモを、将来的に釣れそう度 `SCORE` へ反映するための設計メモです。

現時点では、外部釣果メモは `SCORE未計算 / SCORE未反映` として扱います。このドキュメントは将来実装時の判断材料を整理するものであり、今回の変更ではSCORE計算ロジック、UI、データ構造、API、DB連携は変更しません。

## 基本方針

- 外部メモは当面 `acquisitionMethod: "manual"` のものだけをSCORE反映候補にします。
- `manualOnly` / `referenceOnly` / `unknown` の情報元は自動収集対象にしません。
- 現時点で `crawlPolicy === "allowed"` の情報元はない前提を維持します。
- 記事本文全文、画像、コメント全文、投稿者プロフィール詳細は保存しません。
- 保存・利用する情報は、魚種、釣果日、エリア、釣り方、匹数、サイズ、出典URL、情報元名、信頼度などの構造化された事実情報に限定します。
- 外部メモは釣果を保証する根拠ではなく、説明可能な参考情報として扱います。
- 外部メモだけで極端に高いSCOREにならないよう、上限や重みを設ける前提にします。
- 将来実装時も、まずは複雑な機械学習ではなく、説明可能なルールベースに留めます。

## SCORE反映候補にする前提条件

外部メモをSCOREへ使う場合は、最低限以下を満たすものだけを候補にします。

| 項目 | 方針 |
| --- | --- |
| 取得方法 | 当面は `acquisitionMethod: "manual"` のみ。`auto` や `ai_assisted` は別Issueで安全性とレビュー状態を設計するまで対象外。 |
| 出典 | `sourceId`、`sourceName`、`sourceUrl` のように、情報元とURLを確認できること。 |
| 信頼度 | `confidence` を必ず考慮し、`low` は強く反映しない。 |
| 釣果日 | 日付が不明なメモは鮮度評価できないため、原則SCORE反映対象外または非常に弱い参考にする。 |
| 魚種 | アプリ内の魚種またはカテゴリに対応付けできること。曖昧な魚種は弱い参考に留める。 |
| 地点 | `spotId` または `areaName` のどちらかを持つこと。未紐づけメモの自動位置推定はしない。 |
| 保存範囲 | 本文全文、画像、コメント全文、プロフィール詳細を使わず、構造化済みの事実情報だけを使うこと。 |

## 地点の扱い

外部メモは地点精度の差が大きいため、`spotId` と `areaName` を分けて扱います。

| 地点情報 | 将来の扱い |
| --- | --- |
| `spotId` あり | 紐づいた釣り場の参考情報として扱う。対象釣り場のSCOREに比較的強く反映できる候補。 |
| `spotId` なし / `areaName` あり | エリア単位の弱い参考情報として扱う。特定釣り場に直接強く加点しない。 |
| `coordinatePrecision: exact` | 許可済み・構造化済みの具体地点として扱える場合のみ、強めの地点一致候補にする。 |
| `coordinatePrecision: approximate` | 推定を含むため、地点一致の重みを下げる。 |
| `coordinatePrecision: rounded` | 公開保護や丸めを含む代表点として扱い、ピンポイント一致とはみなさない。 |
| `coordinatePrecision: unknown` | 地点加点は原則行わず、エリア参考に留める。 |

公開範囲を広げる場合は、釣り場保護、混雑防止、安全配慮のため、詳細座標の丸め、非公開化、危険な小場所の除外方針を維持します。

## 加点候補と重み付け方針

将来のSCORE反映では、外部メモ由来の加点を小さな補助要素として扱います。数値は実装時に別Issueで調整しますが、重みの方向性は以下とします。

| 観点 | 加点候補 | 弱める・反映しない候補 |
| --- | --- | --- |
| `confidence` | `high` は強め、`medium` は中程度。 | `low` は弱い参考または理由表示のみ。 |
| 魚種一致 | 対象魚種と完全一致する直近メモは加点候補。`青物`、`根魚` などカテゴリ一致も候補。 | 魚種不明、カテゴリが広すぎる、対象魚種とずれる場合は反映しない。 |
| 釣り場一致 | `spotId` が同じ場合は加点候補。 | `areaName` のみの場合は弱める。地点不明は反映しない。 |
| エリア一致 | 同じ `areaName` の直近メモは弱〜中程度の加点候補。 | 隣接しないエリアや広すぎるエリア表現は弱める。 |
| 釣り方一致 | ルアー/エサ、エギング、サビキ、ショアジギングなどが近い場合は加点候補。 | 釣り方不明、対象魚種と釣法の関連が薄い場合は弱める。 |
| サイズ | 対象魚種として良型のサイズが構造化されている場合は加点候補。 | サイズ不明、誇張表現のみ、本文由来の未構造化情報は使わない。 |
| 匹数 | 複数匹、まとまった釣果などが構造化されている場合は加点候補。 | 匹数不明、単発釣果、表現が曖昧な場合は弱める。 |
| 鮮度 | 直近釣果ほど強く扱う。 | 古い釣果は日数経過で影響を弱め、一定期間を超えたものは対象外または理由表示のみ。 |
| 情報元 | レビュー済みで信頼できる手動メモは候補。 | `manualOnly` / `referenceOnly` / `unknown` を自動収集したかのように扱わない。 |

## 鮮度の比較方針

鮮度は、釣果日と現在または予測対象時点の差分で評価します。

- 直近の外部メモほど強く扱います。
- 一定日数を超えた外部メモは、SCORE反映対象外または弱い参考にします。
- 魚種ごとの季節性を将来拡張できるよう、単純な日数減衰だけに固定しない設計にします。
- 古いが同じ季節の実績として価値があるメモは、別枠の「季節参考」として扱う余地を残します。
- 日付が不明、または釣果日ではなく投稿日しか分からない場合は、安全側に倒して弱く扱います。

## 釣果当時と現在/予測時点の条件一致

時間帯、潮汐、天候、風、波、海水温などは、将来の加点候補として扱えます。ただし、いずれも参考情報であり、安全判断、航行判断、公式潮汐表の代替、釣果保証には使いません。

### 時間帯

最初は細かい時刻一致ではなく、粗い区分から始めます。

| 区分例 | 比較方針 |
| --- | --- |
| 朝まずめ | 釣果当時と予測時点が同じ区分なら加点候補。 |
| 日中 | 明るい時間帯の大まかな一致として扱う。 |
| 夕まずめ | 朝まずめとは別区分として扱う。 |
| 夜 | 夜釣り対象魚や釣法で参考にする。 |
| 不明 | 時間帯一致による加点はしない。 |

条件が大きく違う場合、たとえば夜の釣果を日中予測へ強く反映することは避け、加点を弱めるか反映しません。

### 潮汐・潮位参考値

潮汐は、まず粗い状態区分と参考値の近さで比較します。

| 観点 | 比較方針 |
| --- | --- |
| 満潮前後 | 釣果当時と予測時点が同じ満潮前後なら加点候補。 |
| 干潮前後 | 同じ干潮前後なら加点候補。 |
| 上げ潮 / 下げ潮 | 潮の動きが同じ方向なら加点候補。 |
| 潮位差 | 大きな潮位差、小さな潮位差などの粗い一致を候補にする。 |
| `sea_level_height_msl` | Open-Meteo Marine APIを使う場合も、潮位そのものを保証する値ではなく潮位参考値として扱う。 |

潮汐データは釣行前の参考情報であり、公式潮汐表、安全判断、航行判断の代替にはしません。潮汐条件が近いことだけで釣れる、または安全であると示す表現は使いません。

### 天候・風・波・海水温

天候・海況は、既存の環境データで扱える範囲から段階的に比較します。

| 観点 | 比較方針 |
| --- | --- |
| 天候 | 晴れ、曇り、雨などの粗い区分が近い場合は加点候補。 |
| 風速 | 弱風、中程度、強風のような粗い区分から始める。強風時の安全判断には使わない。 |
| 風向 | 追い風、向かい風、横風など釣り場との関係を将来検討できるようにする。 |
| 波高 | 低い/中程度/高いなどの参考区分に留める。荒天時の釣行可否判断には使わない。 |
| 海水温 | 魚種ごとの季節性と組み合わせる候補にする。 |

釣果当時と現在/予測時点の条件が大きく違う場合は、外部メモ由来の加点を弱めるか、SCORE反映せず理由表示の候補に留めます。

## SCORE根拠表示の方針

将来SCOREへ反映する場合は、外部メモ由来であることを理由表示で分かるようにします。モック釣果由来、環境データ由来、外部メモ由来は区別できる設計にします。

表示文言の例:

- `手動登録された外部釣果メモに同魚種の直近情報があります。`
- `同じエリアの手動外部メモに近い釣り方の釣果があります。`
- `釣果当時と予測時点の時間帯が近い外部メモがあります。`
- `釣果当時と予測時点の潮位参考値が近い外部メモがあります。`
- `釣果当時と予測時点の天候・海況条件が一部近い外部メモがあります。`

理由表示では、以下を避けます。

- 外部サイト本文や画像を根拠として転載すること。
- `必ず釣れる`、`安全に釣行できる`、`潮位が正確` などの保証表現。
- 出典不明または信頼度の低いメモを、確定情報のように表示すること。

## 安全側の制約

- 外部メモだけでSCOREを大きく押し上げないよう、外部メモ由来の加点上限を設ける前提にします。
- `confidence: low`、地点不明、日付不明、魚種不明のメモは強く反映しません。
- 未紐づけメモの自動位置推定はしません。
- 自動収集を前提にしたルールにはしません。
- 潮汐、天候、海況の一致だけで釣果を保証する表現はしません。
- 潮汐、天候、海況データを安全判断、航行判断、荒天時の釣行可否判断には使いません。
- 新しい潮汐API、天候API、有料APIは、この設計だけでは追加しません。

## 今回の対象外

このドキュメント追加では、以下を行いません。

- SCORE計算ロジックの実装変更。
- 外部釣果メモのSCORE反映本実装。
- UI変更。
- スクレイピング。
- 外部サイトへのHTTPアクセス。
- 定期実行ジョブ。
- AI解析。
- DB/Supabase連携。
- localStorage key の変更。
- 外部釣果メモのデータ構造変更。
- 未紐づけメモの自動位置推定。
- 新API追加。
