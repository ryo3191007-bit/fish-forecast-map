## 1. 比較ルール

- 各AIへ同じプロンプト、同じSchema、同じ代表座標を渡す
- 出力を人間が修正する前に原本を保存する
- Schemaエラーも品質評価へ含める
- 値の一致数だけでなく、根拠の質と `unknown` の適切さを評価する
- もっとも多く埋めたAIを自動的に高評価しない
- 規制・施設・水深等を弱い根拠で断定した場合は減点する

共通プロンプト:

```txt
docs/research/KARATSU_EAST_PORT_MULTI_AI_PROMPT.md
```

## 4. レビュー観点

### 4.1 重大な問題

次があれば再調査対象です。

- 民間釣りサイトの文章・魚種一覧・地図ピンを実質的に転記
- 港湾施設の存在から、立入可能・釣り可能と断定
- 公式情報が見つからないことを「禁止なし」と判定
- 港湾照明や標識灯を常夜灯として断定
- 複数の施設水深から、釣り位置の水深を根拠なく選択
- 出典URLが属性を実際には支えていない
- `confirmed` と `inferred` の混同
- 不明項目をもっともらしい値で補完

### 4.2 判定差が出たとき

1. sourceの信頼度を比較する
2. sourceが同じ対象範囲を指すか確認する
3. 地区全体と具体施設を混同していないか確認する
4. 情報の公開日・確認日を比較する
5. それでも決められなければ `unknown` を採用する

## 5. Post-MVP-039仕様へのフィードバック候補

- `scopeType`: district / facility / access_point

- 施設の存在と釣り目的利用可否の分離
