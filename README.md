# fish-forecast-map

釣果予測マップは、福岡県糸島市西岸から唐津湾、伊万里湾、平戸方面までの陸っぱり釣り情報を地図で確認するための個人利用Webアプリです。

主な利用者はリポジトリ所有者本人です。公開サービスとしての利用は想定しません。

対象は、堤防・磯場などの陸っぱり釣りです。釣法はルアー・エサの両方を扱います。

最初のバージョンでは、モックデータを使って低コストなMVPを作ります。後続フェーズで、潮汐・天気・水温データ連携、3D海底地形表示、釣果予測スコアの高度化、釣具店等の釣果情報取り込みを追加する想定です。

## MVP v0.1

MVP v0.1では以下を提供します。

- Next.js + TypeScriptのWebアプリ。
- MapLibre GL JSによる地図画面。
- 糸島西岸から唐津湾、伊万里湾、平戸方面までのモック釣果データ。
- 地図上の釣果地点マーカー。
- 釣果情報一覧。
- 魚種フィルタ。
- 0〜100点の釣れそう度スコア。
- スコア根拠の表示。
- 釣れそう度は釣果を保証するものではなく、参考情報であることの注意書き。

## MVP v0.1 ではやらないこと

- 釣果登録フォーム。
- ユーザー自身の釣果登録。
- 外部釣果サイトの自動取り込み。
- 記事URL貼り付け機能。
- Supabase/DB連携。
- 潮汐・天気・水温データ連携。
- 3D海底地形表示。
- 実績ベースの高度な予測。
- ログイン/認証。
- 有料APIの利用。

## コスト方針

以下の順に優先します。

1. 既存のChatGPT Plus利用範囲を中心に、月額コストを抑えること。
2. トークン消費を抑えること。
3. 品質を維持すること。
4. 生産性を高めること。

明示的な承認がない限り、有料API、有料ホスティング、有料地図サービス、有料モデル推論は導入しません。

## 開発フロー

- ChatGPTが司令塔/プロジェクトリードとして動きます。
- CodexはGitHub IssueとPull Requestを単位に実装担当として動きます。
- 1タスク = 1 GitHub Issue = 1 Pull Request とします。
- 仕様書、AGENTS.md、README、設計ドキュメントの作成・更新は、必ずこのチャットでユーザー確認後にGitHubへ反映します。
- Issue作成はChatGPTが行ってよいです。
- Codex起動はユーザー承認後に行います。
- PRマージは必ずユーザーの最終承認後に行います。

詳細は `AGENTS.md` と `docs/CODEX_WORKFLOW.md` を参照してください。

## データ利用上の注意

釣具店、ブログ、SNS、ニュースサイトなどの釣果情報を扱う場合は、利用規約、robots.txt、著作権、アクセス負荷に配慮します。

MVPでは外部サイトの取り込みは行わず、モックデータのみを使います。将来的に取り込む場合も、保存するのは日付、魚種、場所、釣果数、サイズ、釣り方、出典名、出典URLなどの構造化データを中心にします。
